# Gemini Voice Assistant - Голосовой Ассистент для Windows

**Современный голосовой ассистент** с автоматическим распознаванием речи и обработкой текста через AI, разработанный на Python с использованием PySide6.

## 📋 Описание

Gemini Voice Assistant — это десктопное приложение для Windows, которое позволяет преобразовывать голосовой ввод в текст с последующей обработкой через Google Gemini AI. Приложение работает в фоновом режиме и активируется по горячим клавишам.

### Основные возможности

- **Распознавание речи** через Faster Whisper (модели small и medium)
- **Обработка текста** через Google Gemini 3.0 / 2.5 (Flash/Pro)
- **Поддержка Gemini 3.0** с автоматическим откатом на 2.5 при недоступности
- **Два режима записи**: обычный и непрерывный (до 15 секунд)
- **Горячие клавиши**: Win+Shift и F1 для быстрой активации
- **Режимы thinking**: переключатели High/Low для Gemini 3 и Thinking/Быстрый для Gemini 2.5 Flash
- **Голосовые профили и модели**: первое слово диктовки включает профиль («Диктовка»/«Ассистент»), следующее слово («про»/«флеш») выбирает модель; работает даже через паузы и тире
- **Поддержка VLESS VPN** для обхода блокировок
- **Настраиваемый интерфейс** с компактным и развернутым режимами
- **Автозапуск с Windows** и работа в системном трее
- **Отображение версии** из файла VERSION в окне и трее
- **История записей** с возможностью просмотра и копирования
- **Профили промптов Gemini**: создавайте, копируйте и переключайтесь между разными инструкциями для обработки текста
- **Защита от тишины и шумов**: Silence guard, встроенный Faster-Whisper VAD и тонкая настройка порогов Whisper для борьбы с галлюцинациями и повторениями

## ✏️ Профили промптов Gemini

В стабильной сборке уже включены две преднастроенные инструкции:

- **«Диктовка»** — профиль для классического режима: ассистент приводит сырую речь к литературному тексту, сохраняет род/падеж, не отвечает на вопросы и не выполняет команды, а также может перевести результат, если диктовка завершается фразой «переведи на … язык». Этот профиль выбран по умолчанию.
- **«Ассистент»** — профиль для продвинутого сценария с мета-командами: ассистент отслеживает рабочий буфер одной сессии, понимает команды «не пиши», «удали», «сделай списком» и т.д., а итоговый текст выдаёт только после команды «вставляй».

Дополнительно вы можете создавать собственные профили, дублировать и переименовывать имеющиеся, чтобы быстро переключаться между нужными инструкциями прямо из окна программы.

## 🎯 Особенности

### Режимы работы

1. **Обычный режим**: записывает речь, пока удерживается клавиша
2. **Непрерывный режим**: записывает 15-секундные сегменты автоматически

### Активационные слова

Все команды можно произносить в любом регистре (например, `ПРО`, `про` или `Про`). По умолчанию доступны:

- **"про"** — использование Gemini 3.0 Pro Preview для сложных задач (с откатом на 2.5 Pro -> 2.5 Flash)
- **"флеш"** — использование Gemini 2.5 Flash для быстрой обработки
- **"выделить"** — использование текста, заранее скопированного в буфер обмена, и отправка его в Gemini Pro (без указания модели по умолчанию используется Pro)

### Контроль качества записи и анти-галлюцинации

Whisper чувствителен к шуму и может «придумывать» текст, если на входе только фон. Для этого в настройках появился блок **Качество записи**:

- **Silence guard** — программный гейт перед распознаванием. Если средний уровень сигнала ниже `Min level` и держится дольше `Min duration (ms)`, буфер просто отбрасывается и в лог попадает предупреждение.
- **Faster-Whisper VAD** — встроенный в faster-whisper детектор голосовой активности; он вырезает паузы и шум ещё до декодирования.
- **`no_speech` / `logprob`** — нативные пороги Whisper. При высокой вероятности токена `<|nospeech|>` и низком среднем `logprob` сегмент считается тишиной. Значения 0.85 и −1.20 делают модель осторожнее, но их можно ослабить для тихой речи.
- **Keep Whisper context** — переключает `condition_on_previous_text`. В выключенном состоянии каждый прогон Win+Shift/F1 независим (старая фраза не «прилипает» к новой), а включённый режим позволяет вести длинные диалоги, но повышает риск повторов.

Стартовые настройки, с которых стоит начать: `Min level` 500, `Min duration` 600 мс, VAD включён, `no_speech` 0.85, `logprob` −1.2, контекст выключен. Подбирайте значения под свой микрофон — изменения применяются мгновенно.

### Автоматический откат моделей

Приложение поддерживает интеллектуальную систему отката (fallback) для обеспечения стабильности работы. Если выбранная модель недоступна (ошибка 403, 429, 500, 503 и т.д.), ассистент автоматически попробует следующую по приоритету:

1. **Gemini 3.0 Pro Preview** (основная для режима "Про")
   ↓ *если недоступна*
2. **Gemini 2.5 Pro**
   ↓ *если недоступна*
3. **Gemini 2.5 Flash**

Это гарантирует, что вы получите результат даже при временных проблемах с новейшими моделями.

## 📦 Установка

### Системные требования

- **OS**: Windows 10/11
- **Python**: 3.8 или выше
- **RAM**: минимум 4 GB (рекомендуется 8 GB)
- **Диск**: ~2 GB свободного места для моделей

### Шаг 1: Установка Python зависимостей

```bash
pip install -r requirements.txt
```

### Библиотеки (для справки):

```

```

### Шаг 2: Установка FFmpeg

FFmpeg необходим для обработки аудио файлов:

1. Скачайте FFmpeg с [официального сайта](https://ffmpeg.org/download.html)
2. Распакуйте архив
3. Добавьте путь к `ffmpeg.exe` в переменную среды PATH

### Шаг 3: ⚠️ ВАЖНО - Установка моделей Faster Whisper

**КРИТИЧЕСКИ ВАЖНО**: Модели Faster Whisper **НЕ ЗАГРУЖАЮТСЯ АВТОМАТИЧЕСКИ**. Их нужно скачать вручную и поместить в папку `whisper_models`.

#### Скачивание моделей:

1. Есть папка `whisper_models` рядом с исполняемым файлом:

```
...Gemini_Voice_Assistant\whisper_models\
```

2. Скачайте модели вручную:
   - **Модель Small**: https://huggingface.co/guillaumekln/faster-whisper-small
   - **Модель Medium**: https://huggingface.co/guillaumekln/faster-whisper-medium
3. **Структура папок должна быть ТОЧНО такой**:

```
whisper_models/
├── faster-whisper-small/
│   ├── config.json
│   ├── model.bin
│   ├── vocabulary.txt
│   └── ...
└── faster-whisper-medium/
    ├── config.json
    ├── model.bin
    ├── vocabulary.txt
    └── ...
```

4. **Проверка перед компиляцией**: если вы планируете собрать собственный `.exe`, заранее поместите скачанные модели в `whisper_models/` в корне проекта. Скрипт `post_build.py` автоматически копирует эту папку рядом со скомпилированной программой, поэтому все модели будут доступны без дополнительных ручных действий.

5. **Проверка в приложении**: после запуска убедитесь, что во вкладке «Основные» отображается активная модель (small или medium).

**Без правильно установленных моделей приложение НЕ СМОЖЕТ распознавать речь!**

### Шаг 4: Получение API ключа Gemini

1. Перейдите на [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Создайте новый API ключ
3. В приложении откройте **Настройки → Gemini → API ключ**
4. Вставьте полученный ключ

## ⚙️ Конфигурация

### Первичная настройка

1. **Запустите приложение** — появится иконка в системном трее
2. **Откройте настройки** — нажмите на кнопку ▼ в главном окне
3. **Выберите микрофон** — вкладка "Аудио"
4. **Выберите модель Whisper** — вкладка "Основные" (small или medium)
5. **Введите API ключ Gemini** — вкладка "Gemini"
6. **Настройте VPN (опционально)** — вкладка "VPN"

### Настройка VLESS VPN

Если Google Gemini API заблокирован в вашем регионе:

1. Получите VLESS URL от вашего VPN-провайдера
2. Откройте **Настройки → VPN**
3. Включите "Использовать VLESS VPN"
4. Вставьте VLESS URL (формат: `vless://uuid@server:port?...`)
5. Настройте порт SOCKS5 (по умолчанию 10809)
6. Включите "Автоматически подключаться при запуске"

### Горячие клавиши

- **Win + Shift**: Запись в обычном или непрерывном режиме (настраивается)
- **F1**: Запись в обычном или непрерывном режиме (настраивается)

## 🔧 Настройка промпта Gemini

Вы можете настроить поведение Gemini для обработки текста:

1. Откройте **Настройки → Gemini**
2. Выберите профиль промпта из выпадающего списка и при необходимости создайте новый (`+`), переименуйте (`✎`) или удалите (`🗑`) существующий. Текст каждого профиля хранится отдельно и автоматически подставляется в поле ниже.

## 🛠️ Компиляция в .exe

### Сборка через Build Tools

В репозитории есть папка `Build_Tools/` с готовым spec-файлом, GUI-компилятором и нужными hook-скриптами. Это гарантирует корректное включение библиотек (ctranslate2, onnxruntime, requests и т.д.).

1. Установите PyInstaller: `pip install pyinstaller`.
2. Запустите `Build_Tools/SpecCompiler.pyw` (двойной клик из Проводника).
3. В поле выбора укажите `Build_Tools/Gemini_Voice_Assistant.spec`.
4. Нажмите **«Скомпилировать»** — после успешного запуска `pyinstaller` автоматически выполнится `post_build.py`, который перенесёт сборку в корень и очистит временные папки.

#### Альтернативно через CLI

Можно обойтись без GUI и выполнить:

```bash
pip install pyinstaller
pyinstaller Build_Tools/Gemini_Voice_Assistant.spec
```

Команда использует те же hook-и и `post_build.py`, поэтому результат идентичен GUI-сборке.

## 📝 Использование

### Базовый workflow

1. Подготовьте окно, куда хотите вставить результат, и установите курсор в нужном месте (ничего копировать не нужно).
2. **Нажмите Win+Shift** и начните говорить.
3. **Отпустите клавиши** — начнется обработка распознанного текста.
4. По завершении ассистент автоматически вставит ответ в то место, где стоял курсор.

### Работа с буфером обмена

1. Выделите текст и вручную скопируйте его (`Ctrl+C`). Программа не делает это автоматически.

2. После копирования нажмите **Win+Shift** и произнесите команду, начинающуюся со слова **"выделить"**.
   
   Пример:
   
   1) Сначала выделите вопрос «Столица Венесуэлы?» и скопируйте его.
   2) Нажмите Win+Shift.
   3) Скажите «выделить флеш ответь на вопрос».
   
   В результате:
   
   - ассистент возьмёт текст из буфера («Столица Венесуэлы?»);
   - выберет модель Flash (по словам «выделить флеш»);
   - отправит в Gemini запрос «ответь на вопрос» с этим текстом;
   - вставит ответ («Каракас») в то место, где находился курсор.

3. Если нужно использовать Pro-модель, произнесите «выделить про …». Если хотите быстро обработать обычным режимом — произносите «выделить [команда]» без дополнительных слов.

### Использование Pro модели

1. Нажмите **Win+Shift**
2. Скажите: **"Про [ваш запрос]"**
3. Запрос будет обработан моделью Gemini 2.5 Pro

### Управление микрофонами (niccmd/nircmd)

В папке с программой находится утилита `nircmd.exe`. Она используется для системных функций, включая возможность переключения микрофона по умолчанию "на лету". Это полезно, если у вас несколько устройств ввода и вы хотите быстро менять их без захода в настройки Windows.

## 🐛 Решение проблем

### Приложение не распознает речь

- ✅ Убедитесь, что модели Whisper загружены в папку `whisper_models/`
- ✅ Проверьте правильность структуры папок моделей
- ✅ В настройках выберите корректный микрофон
- ✅ Проверьте уровень громкости в индикаторе

### Ошибка Gemini API

- ✅ Проверьте правильность API ключа в настройках
- ✅ Если API заблокирован — настройте VLESS VPN
- ✅ Проверьте подключение к интернету
- ✅ Убедитесь, что порт прокси доступен
- ✅ При ошибке `403 Forbidden` приложение автоматически переключится на модели Gemini 1.5. Также можно вручную выбрать доступную модель во вкладке **Gemini**.
- ✅ Если Pro-модель недоступна, в месте курсора появится сообщение «Gemini Pro недоступна или не отвечает…». В этом случае повторите команду с ключевым словом «флеш».

### VPN не подключается

- ✅ Проверьте корректность VLESS URL
- ✅ Убедитесь, что `xray.exe` присутствует в папке
- ✅ Проверьте доступность порта SOCKS5
- ✅ Просмотрите логи приложения

### Низкое качество распознавания

- ✅ Используйте модель **medium** вместо small
- ✅ Улучшите качество микрофона
- ✅ Говорите четко и ближе к микрофону
